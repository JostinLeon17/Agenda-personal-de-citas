## **Qué es Markdown**

Markdown es un lenguaje de marcado ligero,su principal objetivo es que el texto plano sea lo más legible posible, tanto en su forma de entrada que es el codigo como una vez renderizado que es lo que se muestra. Permite a los usuarios escribir contenido utilizando una sintaxis sencilla y fácil de entender —como asteriscos para negritas o guiones para listas— que luego se puede convertir en HTML u otros formatos para su visualización. 


Principalmente se usa en proyectos de software por su facil documentacion.



##### **Archivos README**

&nbsp;El archivo readme es algo importante en todos los proyectos ya que es lo primero que el usuario ve, contiene pasos basico de como funciona nuestro proyecto, que hacermos con el o como instalarlo.



##### **Sistemas de Control de Versiones**

&nbsp;Sitios como github presentan archivos .md con formato automaticamente lo que permite que se vean claros y bien organizadoss




## **Ventajas de usar Markdown con GitHub** 

 Los archivos pueden entenderse facilmente como texto simple, pero las plataformas como github las convierten en paginas organizadas y agradables visualmente, ademas, es mucho mas agil redactar documentacion en   markdown que hacerlo con HTML o otros lo que agiliza mucho la creacion de contenido.


#### **Mejor control de versiones**

 Al ser archivos de texto plano, herramientas como git pueden detectar y administrar sus modificaciones de forma rapida y precisa.


## **Desventajas de usar Markdown con GitHub** 

##### **Funcionalidad Limitada**

 Markdown al ser de diseño simple si necesitas funciones o ilustracion mas avanzada tendras que recurrir a lenguajes mas avanzados como HTML u otros.



##### **No ideal para documentos largos**
 
 Cuando la documentacion es enorme como libros o guias extensas Markdown puede volverse dificil de organizar y mantener
